WHAT IS MYSQL?

MySQL is the world's most popular open-source relational database management system (RDBMS). It organizes data into tables (rows and columns) and uses Structured Query Language (SQL) to access it.

Key Concepts:

  • Database: A container for tables.
  • Table: A collection of related data (like a spreadsheet).
  • Row (Record): A single entry of data.
  • Column (Field): A specific attribute of the data.

CRITICAL FIX (YOUR ISSUE) ALTER

How to remove UNIQUE constraint

The error happened because email had a UNIQUE index. To allow duplicates: 

-- 1. Find the index name
SHOW INDEX FROM users;
-- (Look for Key_name, likely 'email')

-- 2. Drop the index
ALTER TABLE users DROP INDEX email;

-- 3. Ensure column allows NULLs (optional)
ALTER TABLE users MODIFY COLUMN email VARCHAR(100) NULL;
DDL (Structure)

Creating Tables

CREATE TABLE employees (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    salary DECIMAL(10,2) DEFAULT 0.00,
    dept_id INT,
    joined_at DATE
);

Modifying Tables

-- Add column
ALTER TABLE employees ADD COLUMN status ENUM('active','inactive');

-- Rename column
ALTER TABLE employees CHANGE COLUMN name full_name VARCHAR(100);
DML (Data)

CRUD Operations

-- Create
INSERT INTO employees (full_name, salary) VALUES ('Harshit', 90000);

-- Read
SELECT * FROM employees WHERE salary > 50000;

-- Update (ALWAYS use WHERE)
UPDATE employees SET salary = salary + 5000 WHERE id = 1;

-- Delete (Careful!)
DELETE FROM employees WHERE id = 5;
JOINS VISUALIZED

Understanding Connections

INNER JOIN: Overlap only. (Users who HAVE orders) 

SELECT u.name, o.amount
FROM users u
INNER JOIN orders o ON u.id = o.user_id;

LEFT JOIN: All from Left (Users), matches from Right. (All users, even with no orders) 

SELECT u.name, o.amount
FROM users u
LEFT JOIN orders o ON u.id = o.user_id;
AGGREGATION

Group By & Having

Use WHERE before grouping, HAVING after grouping. 

-- Count users per country
SELECT country, COUNT(*) as total_users
FROM users
GROUP BY country
HAVING total_users > 100;
SUBQUERIES

Query inside a Query

-- Find users with above average salary
SELECT name, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);
TRANSACTIONS (ACID)

All or Nothing

Critical for payments/banking. Ensures data integrity. 

START TRANSACTION;

UPDATE accounts SET balance = balance - 500 WHERE id = 1;
UPDATE accounts SET balance = balance + 500 WHERE id = 2;

-- If everything is okay:
COMMIT;

-- If error occurs:
ROLLBACK;
INDEXING & PERFORMANCE

Why use Indexes?

Like a book index. Speeds up reads, slows down writes. 

-- Create Index on frequently searched column
CREATE INDEX idx_lastname ON users(lastname);

-- Check performance
EXPLAIN SELECT * FROM users WHERE lastname = 'Tiwari';

Tip: Don't index everything. Only columns used in WHERE, JOIN, or ORDER BY.

TOP 10 INTERVIEW QUESTIONS

1. What is the difference between DELETE, TRUNCATE, and DROP?
  • DELETE: DML command. Deletes rows one by one. Can be rolled back. Slower.
  • TRUNCATE: DDL command. Resets table (removes all data). Cannot be rolled back easily. Faster. Reset auto_increment.
  • DROP: DDL command. Removes the entire table structure and data.
2. Explain ACID properties in a database.
  • Atomicity: All or nothing (Transaction succeeds completely or fails completely).
  • Consistency: Database goes from one valid state to another.
  • Isolation: Concurrent transactions don't interfere with each other.
  • Durability: Once committed, data stays saved even if power fails.
3. What is Normalization? Why do we do it?

Normalization is the process of organizing data to reduce redundancy and improve data integrity.

  • 1NF: Atomic values (no lists in a cell).
  • 2NF: No partial dependency (all columns depend on full Primary Key).
  • 3NF: No transitive dependency (non-key columns don't depend on other non-key columns).
4. Difference between WHERE and HAVING?

WHERE filters rows before aggregation (GROUP BY).
HAVING filters groups after aggregation. 

SELECT city, COUNT(*) FROM users
WHERE active = 1      -- Filter rows
GROUP BY city
HAVING COUNT(*) > 5;  -- Filter groups
5. What is the difference between CHAR and VARCHAR?

CHAR(n): Fixed length. If you store "Hi" in CHAR(10), it pads with 8 spaces. Faster for fixed length data (Postal codes).

VARCHAR(n): Variable length. "Hi" takes only 2 characters + length byte. Saves space.

6. Explain Left Join vs Inner Join.

Inner Join: Returns only rows where there is a match in BOTH tables.

Left Join: Returns ALL rows from the left table, and matching rows from the right. If no match, right side is NULL.

7. What is a Primary Key vs Unique Key?

Primary Key: Uniquely identifies a row. Cannot be NULL. Only one per table.

Unique Key: Ensures values are unique. Can be NULL (usually). Multiple allowed per table.

8. How do you find the second highest salary? 
SELECT MAX(salary) FROM employees
WHERE salary < (SELECT MAX(salary) FROM employees);
-- OR using LIMIT
SELECT salary FROM employees
ORDER BY salary DESC LIMIT 1 OFFSET 1;
